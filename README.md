# shadcn-skill-design-starter

> A Next.js starter that ships with a **Claude Code skill** and a full **shadcn/ui + Tailwind v4** design system synced from Figma.
> Drop it open in Claude Code and the AI builds UI that matches your design tokens — no hex-eyedropping, no theme drift.

## What's in the box

| Layer | What you get |
|-------|--------------|
| **Next.js 16 (App Router)** | TypeScript, ESLint, Turbopack, `app/` at root |
| **Tailwind CSS v4** | `@theme` block + 4 themes (Light / Dark / Primary / Secondary) wired to CSS variables |
| **Design tokens** | 1,807 tokens from Figma (`variables-export.json`) — colors, spacing, radius, typography, opacity |
| **Claude Code skill** | `.claude/skills/shadcn-ui-design/` auto-loads when you ask Claude to build UI |
| **Token sync** | `sync-tokens.py` regenerates `app/globals.css` from the Figma JSON in one command |
| **Fonts** | Inter (sans), IBM Plex Sans Thai (Thai), Geist Mono (mono) via `next/font` |

## Quick start

```bash
# 1. Install
npm install

# 2. Generate globals.css from the design tokens
python .claude/skills/shadcn-ui-design/scripts/sync-tokens.py --output app/globals.css

# 3. Run
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

### Useful commands

```bash
npm run dev      # dev server with Turbopack
npm run build    # production build
npm run start    # serve production build
npm run lint     # ESLint
```

## How Figma → code stays in sync

```
   Figma                    references/                       app/globals.css
   ─────                    ───────────                       ───────────────
[Variables]   ─export→   variables-export.json   ──sync-tokens.py──→   :root
                                                                       .dark
                                                                       [data-theme="primary"]
                                                                       [data-theme="secondary"]
                                                                                ↓
                                                              Components use semantic
                                                              tokens via Tailwind classes
                                                              (bg-background, text-foreground, …)
```

1. Designer exports updated variables JSON from Figma.
2. Replace `.claude/skills/shadcn-ui-design/references/variables-export.json`.
3. Run `python .claude/skills/shadcn-ui-design/scripts/sync-tokens.py --output app/globals.css`.
4. Commit both files — diff in `globals.css` is the reviewable record of token changes.

## How the Claude Code skill works

Open the project in [Claude Code](https://claude.com/claude-code) and ask:

> "Build a settings page with a form for name, email, and a delete-account button."

Claude reads `CLAUDE.md` automatically, the `shadcn-ui-design` skill auto-loads, and the generated UI follows:

- **Colors**: only semantic tokens (`bg-background`, `text-foreground`, `bg-primary`, …)
- **Next.js**: `async` Server Components, `metadata` export, `loading.tsx`, `error.tsx`
- **Forms**: Server Action + `<form action>` for simple cases, `react-hook-form` + `zod` for complex
- **Safety**: `<AlertDialog>` confirmation on destructive actions
- **Accessibility**: WCAG 2.1 AA contrast, focus ring, semantic HTML

Full rules: [`.claude/skills/shadcn-ui-design/SKILL.md`](./.claude/skills/shadcn-ui-design/SKILL.md)
Full token reference: [`.claude/skills/shadcn-ui-design/references/DESIGN.md`](./.claude/skills/shadcn-ui-design/references/DESIGN.md)

## Themes

All four themes are defined in `app/globals.css` and toggled by class/attribute on `<html>`:

| Theme | Selector |
|-------|----------|
| Light (default) | `:root` |
| Dark | `class="dark"` |
| Primary (blue) | `data-theme="primary"` |
| Secondary (yellow) | `data-theme="secondary"` |

Components need no changes between themes — every visual property resolves through CSS variables.
